# Contributing

First off all, thanks for wanting to contribute to my library. I really appreciate it.

The following is a set of guidelines, not hard rules and may not need to be followed in certain cases.

## Making a PR

- Ensure that the PR is focused on a single cohesive change rather than multiple unrelated small changes throughout the codebase.
- AI-assisted PRs are _fine_ as long as the author understands what change is being made and why.  

## Code Formatting

- All contributions to the code must be formatted with [Ruff](https://docs.astral.sh/ruff/).

## Commit Messages

- Commit messages should be formatted in:

```
[FEAT/CHANGES/FIX, ...] Short summary of the changes

- Things Changed 1
- Things Changed 2
```

The points can be omitted if the summary is sufficiently explanatory

For any questions regarding the library and contributing, you can ask me at [the discord server](https://discord.gg/mxQxxxpBsB).
